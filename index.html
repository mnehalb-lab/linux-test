You need to find all files modified in the last 30 minutes under /var/log and list them with human-readable sizes. Which command is correct?

A find /var/log -mmin 30 -exec ls -lh {} \;
B find /var/log -mmin -30 -exec ls -lh {} \;
C find /var/log -mtime -30 -ls
D ls -lhR /var/log | grep "$(date -d '30 min ago')"
Correct: B. -mmin -30 = modified less than 30 min ago. Without minus (-mmin 30) matches exactly 30 min. -mtime works in 24h units, not minutes.
Q02 Linux

A process is consuming excessive CPU. After running ps aux you see PID 4821. You want to reduce its priority without killing it. What is the correct approach?

A kill -STOP 4821
B nice -n 19 4821
C renice +19 -p 4821
D ionice -c 3 -p 4821
Correct: C. renice changes niceness of a running process. nice only sets priority at launch. ionice affects I/O scheduling. kill -STOP pauses entirely.

You run df -h and root / is at 100% but du -sh /* shows only 60% usage. What is the MOST likely cause?

AReserved blocks for root are consuming the difference
BHidden files in the root directory are not counted by du
CThe filesystem is corrupted and needs fsck
DDeleted files still held open by running processes are occupying space
Correct: D. A deleted file whose handle is still open by a process occupies disk space until the process releases it. Use lsof | grep deleted to find them.

Q05 Linux

You need to extract only lines 500–700 from a 10GB log file as fast as possible. Which is the BEST approach?

A sed -n '500,700p' largefile.log
B awk 'NR>=500 && NR<=700' largefile.log
C grep -n '' largefile.log | sed -n '500,700p'
D tail -n +500 largefile.log | head -n 200
Correct: A. GNU sed -n '500,700p' stops after line 700, making it fastest on large files. awk and tail+head both work but continue reading more of the file.
Q06 Linux

After running chmod 4755 /usr/bin/myscript, what does the 4 represent?

ASticky bit — prevents deletion by non-owners
BGID bit — script runs with group owner's privileges
CSUID bit — script runs with file owner's effective UID
DRead-only lock on the binary
Correct: C. 4 = SUID (Set User ID). The process runs with the file owner's UID. 2 = SGID, 1 = Sticky bit.
Q07 Linux

You want to watch /var/log/syslog in real time but also filter only lines containing "ERROR" or "CRITICAL". Which command achieves this?

A tail -f /var/log/syslog | grep ERROR CRITICAL

In a Dockerfile, what is the key difference between COPY and ADD?

AADD is faster because it uses parallel transfers
BADD can fetch remote URLs and auto-extract tar archives; COPY only copies local files
CCOPY preserves file permissions; ADD does not

Q09 Docker

A container exits immediately. To debug it, you want to start it interactively and override the default entrypoint. Which command is correct?

A docker run -it myimage /bin/bash
B docker exec -it mycontainer /bin/bash
C docker run -it --entrypoint /bin/bash myimage
D docker start -ai mycontainer
Correct: C. --entrypoint overrides the ENTRYPOINT instruction. Option A appends bash as a CMD argument to the existing entrypoint. Option B requires a running container.
Q10 Docker

You have a multi-stage Dockerfile. Stage 1 is named builder. How do you copy only the compiled binary from stage 1 into the final stage?

A COPY --from=builder /app/mybin /usr/local/bin/

C ADD --from=builder /app/mybin /usr/local/bin/

What does docker system prune -a --volumes remove?

AOnly stopped containers and dangling images
BAll stopped containers, ALL unused images (not just dangling), unused networks, and unused volumes
CAll containers (including running ones) and all images

Correct: B. -a includes ALL unused images (not just dangling). --volumes adds unused named volumes. Running containers and their images are never removed.
Q13 Docker

In Docker Compose, api depends on db. After adding depends_on: [db], the api still fails because the database isn't ready. What is the correct fix?

AAdd restart: always to the api service
BAdd links: [db] to the api service

Q14 Docker

You want to limit a container to use at most 1.5 CPUs and 512MB of RAM. Which docker run flags are correct?

A --cpus="1.5" --memory="512m"
B --cpu=1.5 --ram=512m

You need to capture only TCP packets on port 443 destined for 10.0.0.5 and save to a pcap file. Which tcpdump command is correct?

A tcpdump tcp port 443 dst 10.0.0.5 -w out.pcap
B tcpdump -w out.pcap 'tcp and port 443 and dst host 10.0.0.5'
C tcpdump -f out.pcap port 443 and host 10.0.0.5
D tcpdump --filter="tcp:443 dst:10.0.0.5" -o out.pcap
Correct: B. tcpdump uses BPF filter syntax. dst host filters by destination IP. Quote the filter expression when combining with boolean operators to avoid shell interpretation.

A server has IP 192.168.10.50/26. What is the broadcast address of this subnet?

A192.168.10.127
B192.168.10.63
C192.168.10.255

Q17 Networking

You want to add a persistent static route: traffic to 10.20.0.0/16 should go through gateway 192.168.1.254 on a modern Ubuntu system using Netplan. What is the correct approach?

A route add -net 10.20.0.0/16 gw 192.168.1.254
B In /etc/network/interfaces: up route add -net 10.20.0.0/16 gw 192.168.1.254
C In /etc/netplan/*.yaml: routes: [{to: 10.20.0.0/16, via: 192.168.1.254}]
D ip route add 10.20.0.0/16 via 192.168.1.254 added to /etc/rc.local
Correct: C. Modern Ubuntu uses Netplan. Routes defined in /etc/netplan/ survive reboots. ip route add is ephemeral. /etc/network/interfaces is not used by default on Netplan systems.
Q18 Networking

You need to test if UDP port 5353 is reachable on host 10.0.0.1. Which tool/command is appropriate?

A nc -vzu 10.0.0.1 5353
B telnet 10.0.0.1 5353
C nmap -p 5353 10.0.0.1
D ping -p 5353 10.0.0.1
Correct: A. nc -u uses UDP, -z is scan mode, -v verbose. telnet is TCP-only. nmap without -sU defaults to TCP. ping uses ICMP with no port concept.
Q19 Networking

Using iptables, you want to rate-limit incoming SSH connections to 3 per minute per source IP (brute-force protection) while allowing established sessions to continue. What is the correct approach?

A iptables -A INPUT -p tcp --dport 22 -m limit --limit 3/min -j ACCEPT
BUse the recent module to track NEW connections per source IP and drop when hitcount reaches 4 within 60 seconds
C iptables -A INPUT -p tcp --dport 22 --limit 3/min -j DROP
D iptables -I INPUT -p tcp --dport 22 -m connlimit --connlimit-above 3 -j DROP
Correct: B. The recent module tracks per-source-IP connection attempts. --hitcount 4 --seconds 60 drops the 4th+ NEW connection per minute per IP. Option A uses a global rate limit, not per-IP. Option D limits concurrent connections, not rate.
Q20 Networking

A web app on port 8080 should only be accessible via localhost. You want to use nginx as a reverse proxy with TLS termination. Which nginx config is correct?

A proxy_pass http://0.0.0.0:8080; inside server { listen 443 ssl; }
B proxy_pass https://localhost:8080; inside server { listen 80; }
C fastcgi_pass 127.0.0.1:8080; inside server { listen 443 ssl; }
D proxy_pass http://127.0.0.1:8080; inside server { listen 443 ssl; ssl_certificate ...; ssl_certificate_key ...; }
Correct: D. TLS terminates at nginx (port 443 ssl) with certificates configured. The backend runs plain HTTP on localhost:8080 — no HTTPS needed between nginx and the local app. fastcgi_pass is for PHP-FPM, not HTTP apps.
